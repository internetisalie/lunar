Reports usage of APIs that are not available inside the Redis/Valkey script sandbox under a Redis or Valkey project target. The following are flagged:
  • io.* — the standard I/O library is completely absent from the sandbox.
  • os.* (except os.time and os.clock) — only the subset declared in the target's bundled os.lua stub is permitted; e.g. os.getenv is blocked.
  • require, dofile, loadfile — module-loading globals are removed from the sandbox.
  • print — standard output is not available; use redis.log for diagnostics.
The allowlist is derived from the bundled per-version stub files, so adding a stub automatically permits the corresponding name. The inspection is a no-op under non-Redis/Valkey targets and ships at WARNING level (RISK-R07) pending live validation.
