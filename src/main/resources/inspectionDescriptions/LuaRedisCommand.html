Reports invalid Redis command usage inside redis.call() and redis.pcall() string-literal command arguments under a Redis or Valkey project target. Three categories:
  • Unknown command — the command name is not in the bundled spec for the target version. A did-you-mean rename quick fix is offered when a close match (Levenshtein distance ≤ 2) exists.
  • Wrong arity — the number of arguments is below the command's required minimum.
  • Nondeterministic before write (Redis 5/6 only) — a command marked nondeterministic (e.g. TIME, RANDOMKEY) is called before a write command without a preceding redis.replicate_commands() guard, violating verbatim replication correctness. Not raised under Redis 7+ or Valkey, which use effects replication.
Dynamic (non-literal) first arguments are never flagged. The inspection is a no-op under non-Redis/Valkey targets.
