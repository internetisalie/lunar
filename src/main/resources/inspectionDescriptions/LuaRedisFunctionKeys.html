Reports usage of KEYS and ARGV globals inside a Redis Function library file (a file starting with #!lua name=<lib>).

These globals are EVAL-only ambient bindings. Redis Functions receive their keys and arguments via the callback parameters (keys, args) passed by the server at invocation time — not via KEYS/ARGV globals, which are absent inside Function callbacks.

Replace usages with the corresponding callback parameter:

redis.register_function('f', function(keys, args)
    local first_key = keys[1]  -- not KEYS[1]
    local first_arg = args[1]  -- not ARGV[1]
end)

The inspection is a no-op under non-Redis/Valkey targets and on files without a #!lua name=<lib> shebang (plain EVAL scripts may use KEYS/ARGV normally). Only Redis 7+ and Valkey targets are affected, as Functions are a Redis 7+ feature.
